## 1. DEFINITIONS

### 1.1 Terms
- **Controller:** Entity determining purpose and means of personal data processing
- **Processor:** Entity processing personal data on behalf of Controller
- **Personal Data:** Information relating to identified or identifiable natural person
- **Processing:** Any operation performed on personal data (collection, storage, use, etc.)
- **Data Subject:** Natural person to whom personal data relates
- **Breach:** Unauthorized disclosure, loss, or corruption of personal data
- **Sub-processor:** Processor engaged by Processor to perform processing activities

### 1.2 Regulatory References
- GDPR: General Data Protection Regulation (EU 2016/679)
- CCPA: California Consumer Privacy Act (Cal. Civ. Code §1798.100 et seq.)
- PIPEDA: Personal Information Protection and Electronic Documents Act (Canada)
- LGPD: Lei Geral de Proteção de Dados (Brazil)
- Other: Applicable data protection laws in Controller's jurisdiction

| **Duration** | [Processing period] |
| **Location** | [Geographical scope of processing] |

### 2.2 Permitted Processing
- Processing shall be limited to purposes specified
- Processing shall be lawful, fair, and transparent
- Processing shall observe data minimization principles
- Processing shall respect data subject rights

### 2.3 Prohibited Processing
- Processing for different purposes without new authorization
- Processing beyond agreed scope
- Processing in manner incompatible with original purposes
- Sale or commercial exploitation of personal data

## 3. PROCESSOR OBLIGATIONS

### 3.1 Processing Instructions
Processor shall:
- Process personal data only on documented written instructions from Controller
- Not process personal data for own purposes
- Not process personal data for different purposes
- Notify Controller if processing instruction violates GDPR or applicable law
- Maintain records demonstrating compliance with instructions

### 3.2 Confidentiality
Processor shall:
- Ensure persons authorized to process data are bound by confidentiality
- Provide similar confidentiality undertakings to sub-processors
- Maintain confidentiality before, during, and after termination
- Implement technical and organizational measures to prevent unauthorized access
- Train all personnel on data protection obligations

### 3.3 Sub-processor Management
Processor shall:
- Obtain prior specific or general written authorization from Controller
- Provide notice of intended sub-processor changes (30 days minimum)
- Allow Controller right to object to sub-processor engagement
- Ensure sub-processors comply with same data protection obligations
- Remain fully liable to Controller for sub-processor performance

### 3.4 Data Protection Compliance
Processor shall:
- Comply with GDPR Articles 28-32 (processor obligations)
- Comply with CCPA if processing California resident data
- Comply with applicable data protection laws
- Implement privacy by design and default principles
- Maintain data protection documentation and records

## 4. DATA SUBJECT RIGHTS

### 4.1 Right to Access
Processor shall:
- Enable Controller to fulfill data subject access requests
- Provide data in structured, commonly used, machine-readable format
- Respond to access requests within 30 days
- Provide data without undue delay

### 4.2 Right to Rectification
Processor shall:
- Enable Controller to correct inaccurate data
- Implement data correction mechanisms
- Notify data subjects of corrections
- Update all records and backups

### 4.3 Right to Erasure ("Right to be Forgotten")
Processor shall:
- Implement secure data deletion capabilities
- Delete data without undue delay upon Controller request
- Delete personal data from all systems and backups
- Verify complete deletion
- Implement "deletion by default" for expired data

### 4.4 Right to Restrict Processing
Processor shall:
- Implement data restriction capabilities
- Restrict processing upon Controller request
- Maintain restricted data but cease active processing
- Lift restrictions only with Controller authorization

### 4.5 Right to Data Portability
Processor shall:
- Provide data in portable, machine-readable format (JSON, CSV, XML, etc.)
- Enable transmission to another processor
- Facilitate direct transmission to data subject if requested
- Support data subject switching between processors

### 4.6 Right to Object
Processor shall:
- Cease processing upon data subject objection (except where required by law)
- Implement objection mechanisms
- Notify Controller of objections
- Document all objections

### 4.7 Rights Related to Automated Decision Making
Processor shall:
- Implement human review of automated decisions affecting rights
- Provide explanation of automated decision logic
- Enable data subjects to request human review
- Implement safeguards against discriminatory decisions

## 5. SECURITY MEASURES

### 5.1 Technical Security Measures
Processor shall implement:

**Encryption**
- ✅ AES-256 encryption of personal data at rest
- ✅ TLS 1.3 encryption of data in transit
- ✅ End-to-end encryption where applicable
- ✅ Cryptographic key management system
- ✅ Secure key storage (HSM/TPM)
- ✅ Regular encryption strength review

**Access Control**
- ✅ Role-based access control (RBAC)
- ✅ Multi-factor authentication (MFA)
- ✅ Principle of least privilege
- ✅ Access logs and monitoring
- ✅ Automatic session timeouts
- ✅ Secure credential management

**Data Protection**
- ✅ Pseudonymization of personal data
- ✅ Anonymization where possible
- ✅ Data minimization (collect only necessary data)
- ✅ Purpose limitation enforcement
- ✅ Secure backup and recovery procedures
- ✅ Data integrity verification

**Network Security**
- ✅ Firewalls and network segmentation
- ✅ Intrusion detection and prevention systems
- ✅ DDoS protection
- ✅ VPN access controls
- ✅ Secure configuration baseline
- ✅ Regular vulnerability scanning

**Application Security**
- ✅ Secure coding practices (OWASP standards)
- ✅ Input validation and sanitization
- ✅ Output encoding
- ✅ Secure authentication and authorization
- ✅ Protection against common vulnerabilities
- ✅ Regular security code reviews

### 5.2 Organizational Security Measures
Processor shall implement:

**Personnel Security**
- ✅ Background checks for personnel with access
- ✅ Data protection training and awareness
- ✅ Confidentiality agreements
- ✅ Clear data protection responsibilities
- ✅ Regular refresher training
- ✅ Incident reporting procedures

**Physical Security**
- ✅ Secure facilities and data centers
- ✅ Access control (badges, biometrics, guards)
- ✅ CCTV monitoring
- ✅ Environmental controls (fire, climate)
- ✅ Visitor management
- ✅ Secure disposal of physical media

**Operational Security**
- ✅ Change management procedures
- ✅ Incident response procedures
- ✅ Business continuity and disaster recovery
- ✅ Regular security testing and audits
- ✅ Penetration testing (at least annually)
- ✅ Security vulnerability disclosure program

**Documentation**
- ✅ Information security policy
- ✅ Data protection policy
- ✅ Security procedures and guidelines
- ✅ Incident response plan
- ✅ Breach notification procedures
- ✅ Regular policy updates

### 5.3 Security Level Determination
Security measures appropriate to:
- Nature of personal data (sensitive vs. non-sensitive)
- Risk of unauthorized processing or disclosure
- Scope and context of processing
- State of art and implementation costs
- Risks to data subjects' rights and freedoms

### 5.4 Security Incident Response
Processor shall:
- Detect security incidents and breaches
- Investigate incidents promptly
- Document incident details
- Notify Controller without undue delay
- Assess impact on data subjects and Controller
- Implement corrective measures
- Cooperate with regulatory authorities

## 6. SUBPROCESSORS

### 6.1 Subprocessor Authorization

Processor shall not engage sub-processors without prior authorization:

**Prior Specific Authorization Required For:**
- New sub-processors
- Significant changes to sub-processor role
- Transfer of personal data to different processor

**General Authorization**
- Controller may grant blanket authorization for future sub-processors
- Notification must precede engagement (30 days minimum)
- Controller right to object remains

### 6.2 Subprocessor Requirements
All sub-processors must:
- Comply with same data protection obligations as Processor
- Provide equivalent security measures
- Sign data processing agreements
- Commit to data subject rights compliance
- Submit to audit and inspection
- Maintain sub-processor agreements in writing

### 6.3 Controller Objection Right
Controller may object to sub-processor engagement:
- On reasonable grounds concerning compliance
- On reasonable grounds concerning security
- Within notification period (30 days minimum)
- Right to terminate affected processing if objection cannot be resolved

### 6.4 Sub-processor Change Management
Processor shall:
- Notify Controller of sub-processor changes 30 days in advance
- Maintain list of current sub-processors
- Provide sub-processor details:
    * Name and location
    * Processing activities
    * Security certifications
    * Contact information
- Update list regularly
- Make list publicly available

## 7. INTERNATIONAL DATA TRANSFERS

### 7.1 Data Transfer Mechanisms

**Adequacy Decisions**
- Transfer to jurisdictions with European Commission adequacy decision
- Examples: Canada, Japan, New Zealand, Israel, UK

**Standard Contractual Clauses (SCCs)**
- Use approved SCCs for transfers to third countries
- Module 1: Controller to Processor
- Module 2: Processor to Sub-processor
- Execute Processor-to-Sub-processor SCCs

**Binding Corporate Rules (BCRs)**
- Where applicable for multinational organizations
- Provide equivalent protection to GDPR
- Approved by supervisory authority

**Supplementary Measures**
- Where standard mechanisms insufficient:
    * Encryption of personal data
    * Pseudonymization
    * Data minimization
    * Confidentiality agreements
    * Regular compliance audits

### 7.2 Transfer Impact Assessment
Processor shall:
- Assess law and practice of destination country
- Evaluate impact on data subject rights
- Implement supplementary measures if necessary
- Document transfer mechanism and assessment
- Update assessment upon legal changes

### 7.3 Transfer Limitation
Processor shall:
- Not transfer data to third countries without authorization
- Comply with data localization requirements
- Restrict transfers to compliant mechanisms only
- Suspend transfers if legal compliance cannot be ensured

## 8. AUDIT AND INSPECTION

### 8.1 Audit Rights
Controller may:
- Audit Processor compliance with DPA
- Inspect Processor facilities and systems
- Review Processor documentation and records
- Conduct on-site audits with reasonable notice
- Engage third-party auditors on Controller's behalf

### 8.2 Audit Frequency
- Initial audit: Upon request
- Regular audits: As determined by Controller risk assessment
- Audit scope: Full or specific areas as required
- Notice period: 30 days minimum for on-site audits

### 8.3 Audit Cooperation
Processor shall:
- Provide reasonable access to facilities
- Provide documentation and records
- Answer audit questions and inquiries
- Implement corrective actions for findings
- Provide status updates on remediation
- Bear reasonable audit costs

### 8.4 Third-Party Certifications
Processor may demonstrate compliance through:
- SOC 2 Type II certification
- ISO 27001 certification
- ISO 27018 certification
- EU Cloud Code of Conduct certification
- Other relevant security certifications
- Certifications reduce audit frequency requirements

### 8.5 Regulatory Inspections
Processor shall:
- Cooperate with supervisory authority inspections
- Provide access to Processor facilities
- Provide documentation and records
- Notify Controller of regulatory action
- Implement required remediation
- Share inspection outcomes with Controller

## 9. DATA BREACH AND INCIDENT RESPONSE

### 9.1 Breach Definition
Breach means:
- Unauthorized access to personal data
- Unauthorized disclosure of personal data
- Unauthorized loss or destruction of personal data
- Accidental or intentional unauthorized disclosure
- Any compromise of confidentiality, integrity, or availability

### 9.2 Breach Detection
Processor shall:
- Implement systems to detect breaches
- Monitor for unauthorized access or activity
- Test detection systems regularly
- Respond to breach alerts promptly
- Investigate suspected breaches

### 9.3 Breach Notification
Processor shall notify Controller **without undue delay** upon discovery:

**Notification Must Include:**
- Breach description and affected data
- Categories and approximate number of data subjects
- Categories and approximate amount of personal data
- Likely consequences of breach
- Measures taken or proposed to address breach
- Contact point for further information
- Impact assessment and remediation status

**Notification Timing:**
- Initial notification: As soon as practical
- Detailed notification: Complete within 72 hours
- Ongoing updates: Regular status updates

### 9.4 Investigation and Documentation
Processor shall:
- Investigate breach thoroughly
- Document investigation findings
- Assess impact on data subjects
- Determine breach severity
- Identify root cause
- Develop corrective measures
- Implement preventive measures

### 9.5 Cooperation with Authorities
Processor shall:
- Cooperate with supervisory authorities
- Provide requested information
- Implement authority recommendations
- Report investigation findings
- Participate in regulatory inquiries

### 9.6 Data Subject Notification
Processor shall:
- Assist Controller with data subject notification
- Provide necessary information for notification
- Draft notification messages if requested
- Facilitate notification communications
- Support data subject inquiries

### 9.7 Media and Public Notification
For large-scale breaches:
- Notify media if required by law
- Make public statements if appropriate
- Coordinate communication with Controller
- Monitor public response and misinformation
- Provide updates as investigation progresses

## 10. TERM AND TERMINATION

### 10.1 Commencement
Agreement becomes effective upon execution by both parties.

### 10.2 Duration
- Continues for duration of processing relationship
- Survives termination of underlying service agreement
- Terminates only when all personal data deleted or returned

### 10.3 Termination for Convenience
- Either party may terminate for convenience with 90 days notice
- Processor continues obligations during notice period
- Processor must delete or return all personal data upon termination

### 10.4 Termination for Cause
- Controller may terminate immediately for material breach
- Material breach includes:
    * Failure to comply with security measures
    * Unauthorized processing or sub-processing
    * Failure to implement data subject rights
    * Failure to cooperate with audits
    * Breach of confidentiality
    * Unauthorized data transfer

### 10.5 Termination Procedures

Upon termination, Processor shall:

**Immediate Actions:**
- Cease all processing of personal data
- Cease new data collection
- Implement data access restrictions
- Preserve all personal data for transition period

**Data Handling:**
- Delete all personal data (default option)
- Return all personal data (if requested by Controller)
- Demonstrate deletion in writing
- Provide certification of deletion or return
- Delete encrypted keys if applicable
- Retain personal data only if required by law

**Post-Termination Obligations:**
- Cooperate with data subject requests
- Support regulatory inquiries
- Maintain audit logs and documentation
- Return all Controller confidential information
- Complete pending projects or transfers

### 10.6 Survival
Following termination:
- Confidentiality obligations survive indefinitely
- Indemnification obligations survive
- Liability obligations survive (to extent allowed)
- Dispute resolution obligations survive
- Regulatory cooperation obligations survive

## 11. GOVERNING LAW

### 11.1 Applicable Law
- Agreement governed by laws of Controller's jurisdiction
- Precedence given to data protection law requirements
- GDPR applies to EU personal data
- CCPA applies to California resident data
- PIPEDA applies to Canadian personal data
- LGPD applies to Brazilian personal data
- Local laws apply to local personal data

### 11.2 Dispute Resolution

**Step 1: Good Faith Negotiation (30 days)**
- Parties attempt resolution through negotiation
- Designate senior representatives
- Document negotiation attempts

**Step 2: Mediation (30 days)**
- If negotiation fails, proceed to mediation
- Select neutral mediator
- Both parties participate in good faith
- Mediation conducted in English or local language

**Step 3: Litigation or Arbitration**
- If mediation fails, proceed to litigation or arbitration
- Venue: Courts of Controller's jurisdiction
- Either party may initiate proceedings
- Prevailing party may recover reasonable attorney fees

### 11.3 Regulatory Authority
Processor acknowledges that:
- Data protection authorities have supervisory powers
- Controller must cooperate with authorities
- Processor must cooperate with authorities
- DPA does not limit regulatory authority jurisdiction
- Regulatory action proceeds independently of disputes
